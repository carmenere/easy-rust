# Errors
## `errno.h`
[**The `errno` man page**](https://man7.org/linux/man-pages/man3/errno.3.html).<br>
The value in `errno` is significant only when the return value of the call indicated an error, i.e., `-1` from most system calls.<br>
The `errno` is **thread-local**; setting it in one thread does not affect its value in any other thread.<br>

<br>

## `EINTR`
**Interruptible functions** may return `EINTR` error.<br>
In POSIX, an **interruptible function** is a *system call* or *library function* that, when **interrupted by a signal** (**without** `SA_RESTART` flag), **returns prematurely** with `-1` and **sets** the global variable **errno** with an *error code* of `EINTR` instead of completing its task. This behavior allows programs to handle asynchronous events (signals) while a function is blocked waiting for an event (e.g., I/O or a timer).<br>

<br>

## `ENETUNREACH` vs. `EHOSTUNREACH`
`ENETUNREACH` means **Network is unreachable**, its code is **101**.<br>
If there is **no route to destination addres in routing table** then kernel returns `ENETUNREACH`.<br>

`EHOSTUNREACH` means **No route to host**, its code is **113**.<br>


Source of error:
- `ENETUNREACH` can often be generated by the local system itself if no suitable network interface or route is found;
- `EHOSTUNREACH` is typically generated by a remote router via an **ICMP message** (probably because the host is down);
